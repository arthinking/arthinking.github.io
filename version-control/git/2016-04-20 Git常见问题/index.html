团队协作问题

如果相同的模块同事开发不同的需求，而拉取了不同的分支，这样会加大代码冲突的可能性，建议的做法是把这些不同需求同一个模块的修改都分配给同一批人，方便模块代码维护和管理。基于模块的开发，后续的版本兼容以前的版本。
